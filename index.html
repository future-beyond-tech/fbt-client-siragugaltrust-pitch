01 — Current State Audit

An honest look at siragugaltrust.futurebeyondtech.in

Before pitching anything, FBT audited every page of the existing site. Here is what works, and here is what's quietly losing donor trust on every visit.

What's working KEEP

  • ✓
    Clean information architecture 8 pages mapped well: Home, About, Programs, Gallery, Team, Donate, Join, Contact
  • ✓
    Emotional copy with Tamil heart "Where Wings Are Given" — strong narrative anchor
  • ✓
    Trust badges present Reg. No. 8/2018, Volunteer-Led, Zero Admin Fee, Community Governed
  • ✓
    4 programs documented Scholarship · Dropout School · Diwali · WoDAC — all present with descriptions
  • ✓
    Razorpay payment link works External, but functional for one-time gifts
  • ✓
    Multi-channel contact Phone · Email · WhatsApp present in footer
  • ✓
    Donation tier suggestions ₹500 · ₹1,000 · ₹5,000 monthly with concrete impact statements

What's broken FIX

  • ✗
    Impact counters all show 0 Lives Impacted: 0 · Scholarship Students: 0 · Dropout Children: 0 · Years: 0
  • ✗
    Donations leave the website External rzp.io link → no analytics, no branded checkout, no receipt UX
  • ✗
    No 80G receipt automation Indian donors expect tax receipts — currently zero infrastructure
  • ✗
    "Download Annual Report" button leads nowhere Transparency promised, transparency artifact missing
  • ✗
    Beneficiary stories are placeholders Three generic quotes — no real names, no faces, no proof
  • ✗
    No financial transparency dashboard Where did last year's ₹X go? No visible answer
  • ✗
    Volunteer signup is a Google Form No volunteer database, no skill tagging, no follow-up
  • ✗
    No Tamil language toggle Tamil Nadu trust, English-only site — accessibility miss
  • ✗
    No CSR partner page Locks out corporate India's mandatory 2% CSR funding pool
  • ✗
    No campaign-level fundraising "Raise ₹3L for 13 students by August" — pattern missing
  • ✗
    No admin dashboard Trust can't update content, donations, stories without a developer
  • ✗
    Site looks static No blog · no events calendar · no recent news — visitors can't tell if you're active

The cost of the gap.

A first-time donor lands on the home page, sees four big counters reading 0, and leaves before reading the mission. Eight years of work — invisible. The Razorpay link becomes a click of last resort, not an invitation. Trust is leaking from a site that's literally about trust.

Live financial dashboard
Updated in real time. Total raised. Total deployed. Per-program allocation. A donor can see exactly where their last ₹500 went without filing an RTI.
Fund-utilization trail
Each donation links to one or more outcome events: a fees-paid receipt, a school-supplies invoice, a hostel-stay log. The chain is auditable.
Beneficiary stories with consent layer
Real names, real photos, real outcomes — but only with documented consent. Stories without consent stay in the audit DB, not on the public site.
Donor portal — your giving, on record
Every donor logs in to see their full history, redownload any 80G receipt, update PAN, and start or pause monthly giving. Tax season, solved.
Quarterly transparency report
Auto-generated. PDF + email + WhatsApp broadcast. Numbers · stories · what went wrong · what's next. Eight years of work, finally documented in public.
05 — Engineering

The stack — chosen for longevity, not novelty.

A trust runs for decades. The website should too. FBT picks tools that compound maintainability and recruit-ability over time, not the framework of the month.

Frontend
Next.js 15
PPR rendering · server components · the React 19 baseline. Fast everywhere, indexable everywhere.
Backend
.NET 8
Vertical Slice + CQRS. Boring · battle-tested · still fast in 2036.
Database
PostgreSQL
Donations, beneficiaries, audit trails. Relational data deserves a relational DB.
Payments
Razorpay
Native embed · UPI + cards + netbanking · webhook-driven receipts.
Auth
Custom JWT + OTP
Donor accounts · admin roles · OTP-before-persistence. No third-party lock-in.
Hosting
Railway + Hostinger DNS
Standalone Node server · DataProtection in DB · zero-downtime deploys.
Analytics
Plausible
No cookies · GDPR-clean · public dashboard option for transparency.
Localization
next-intl
Tamil + English routes · server-rendered · proper diacritic handling.
